# Divide and Conquer Algorithms Implementation

This repository contains the implementation of two fundamental algorithms using a **Divide and Conquer** approach: the **Closest Pair of Points** algorithm and **Integer Multiplication** using the Karatsuba method. It also includes an automated process for generating diverse test datasets for both algorithms.

---

## Summary of Implementation

### 1. Closest Pair of Points Algorithm

| Feature | Details |
| :--- | :--- |
| **Algorithm** | Divide and Conquer approach |
| **Time Complexity** | $O(n \log n)$ |
| **How it Works** | * Divides the set of points into **left and right halves**. * Recursively finds the closest pairs in each half. * Checks the narrow **strip** between the halves for potentially closer pairs. * Uses **brute force** for small subproblems ($\le 3$ points). |

### 2. Integer Multiplication (Karatsuba Algorithm)

| Feature | Details |
| :--- | :--- |
| **Algorithm** | Karatsuba's Divide and Conquer method |
| **Time Complexity** | $O(n^{1.585})$ |
| **Standard Comparison** | Significantly faster than the standard multiplication's $O(n^2)$ |
| **How it Works** | * Splits large numbers into high and low parts. * Uses only **3 recursive multiplications** instead of the standard 4. * Combines results using clever additions and subtractions. |

## Dataset Generation

The code automatically generates a total of **20 test files** (10 for each problem) in a dedicated `datasets/` folder upon execution.

### Closest Pair Datasets (10 files)

* **File Sizes (Number of Points):** 150, 200, 300, 500, 1000, 120, 180, 250, 400, 800 points.
* **Format:**
    1.  First line: The **number of points** ($n$).
    2.  Subsequent $n$ lines: The **x y coordinates** of each point.

### Integer Multiplication Datasets (10 files)

* **Integer Sizes (Number of Digits):** Ranging from **110 to 450 digits**.
* **Format:**
    1.  First line: The **first integer**.
    2.  Second line: The **second integer**.

## How to Use

To run the implementation and generate the datasets:

1.  **Run the code** (e.g., using a command like `python main.py`).

The execution will automatically:
* Test both algorithms with small sample inputs.
* Generate all 20 dataset files in a new directory named `datasets/`.

#### **Generated Files**

The `datasets/` folder will contain:

* `closest_pair_input_1.txt` through `closest_pair_input_10.txt`
* `integer_mult_input_1.txt` through `integer_mult_input_10.txt`
